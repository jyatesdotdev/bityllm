## 2026-07-09 18:20 — Build + integrate the programmatic core (hybrid shell: real code for deterministic commands)
- **What:** Shipped the "real code" half of the hybrid architecture across three new modules and wired it into the live shell:
  - `src/infer/vfs.ts` — an in-memory POSIX-ish virtual filesystem (`~`/`.`/`..` resolution, mkdir -p, write/append, move/copy/remove, a seeded lived-in Debian home).
  - `src/infer/coreutils.ts` — ~35 deterministic binaries as pure functions over the VFS: FS (`ls`/`cat`/`cd`/`pwd`/`mkdir`/`rm`/`mv`/`cp`/`touch`/`chmod`…), identity (`whoami`/`id`/`uname`/`hostname`/`date`/`env`…), text/pipe tools (`grep`/`wc`/`head`/`tail`/`sort`/`uniq`/`tr`/`nl`/`rev`/`seq`), plus a `DREAMED` set naming the commands still left to the model.
  - `src/infer/shell-exec.ts` — a mini-shell: tokenize w/ quotes, pipes `|`, redirects `>`/`>>`, operators `; && ||`, `$VAR`/`$?`/`~`/globs, `VAR=val` assignment. Routes each line to CORE (real) or, if any command isn't core, hands the **whole line** back for the model to dream — with a pre-scan so a dreamed line never half-mutates the VFS.
  - Integrated into `Shell.run` (routing: scripted binaries → programmatic core → model), VFS-backed `cd` (validates against the real tree; a bad `cd` errors and stays put), `Shell.commandNames()` feeding tab-completion, `reboot` resets the real cwd.
- **Why:** The breadth-ceiling diagnosis — the model gibberished outside ~50 commands and the referential-consistency corpus burden (`ll`/`ssh`/`top` bugs, `cat` after `rm`) was structural. Deterministic + stateful + must-be-consistent → real code; generative + variety-is-the-point → the model.
- **Impact:** `ls`/`cat`/pipes/redirects/`&&`/globs are now always correct and consistent (real output fed back into the dream context, capped, so subsequent dreams stay truthful). **48/48 tests green** (+11 new `test/shell-core.test.ts`), typecheck clean, web bundle verified (vfs/coreutils/shell-exec emitted to `dist/`). Four interactive tests were updated to the new (correct) semantics — `cd` now validates against the VFS instead of blind string nav. Next: rebalance the corpus capture-heavy (synthetic shrinks to just the dreamed set), retrain a MINI, eval out-of-band.

## 2026-07-08 21:42 — Train wide 57M model overnight to make WebGPU win in-browser
- **What:** Launched an overnight MLX run — 8L/12H/**768-dim ≈ 57M params**, enlarged 53.5MB v8 corpus, 40k steps (~4.5h on the Mac GPU). Smoke-tested clean first (params confirmed, trains, no NaN).
- **Why:** In-browser token generation is **dispatch-bound** — ~100 tiny kernel launches/token at ~200µs each dominate (the actual matmul math is microseconds), so WebGPU loses to pure-TS CPU for small models. Key lever: the **GPU time is flat vs model width** (fixed dispatch count) while the **CPU time scales with compute** — so *widening* (not deepening — depth adds dispatches) makes the CPU slower while the GPU stays ~48 tok/s, tipping the engine race to WebGPU.
- **Impact:** When done → export int8 (~57MB) as an opt-in **ULTRA** slot in the demo's MODEL knob; WebGPU should beat CPU (~48 vs ~22 tok/s), so the GPU finally gets used. Caveat: "GPU wins" ≠ "GPU fast" — still dispatch-bound; **kernel fusion** (fewer, bigger dispatches) is the separate fix for actual GPU speed.
